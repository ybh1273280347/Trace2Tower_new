# WebShop 人工 Skill 诊断

## 协议

诊断使用 scale-study 固定的 100 个 WebShop test 任务，每任务 3 repeats，共 300 episodes/条件。执行模型统一为 `deepseek-v4-pro`。

人工 skill 绕过 embedding、相似度检索和 Tower/SkillX 展开，原文直接注入 agent 的 `# Retrieved Experience`。resolved config 记录 skill ID、文件路径和 SHA256；单任务 smoke 中记录的 `skill_context_chars` 与文件字符数完全一致，因此直接注入链路已得到验证。

## 解释

V1 强调全面核验和“未知即拒绝购买”，诱发重复搜索、信息页探索和迟迟不购买，因而显著伤害 reward 并增加步骤与 token。V2 将经验改写为有界状态机：一次主搜索、最多一次改写、最多两个候选、已有证据不重复核验、满足条件后立即购买。该改写同时提高 reward 和满分率，并显著减少步骤、无效动作和 chat token。

因此，当前证据不支持“agent 无法加载或执行 skill”。相反，skill 原文能够产生显著正负两种效果，说明内容和展开方式是关键变量。后续优化应优先控制注入上下文中的探索边界、停止条件、重复核验和购买触发，而不是单纯增加轨迹、卡片数量或上下文长度。自动方法仍可能存在检索错配或展开噪声，但需要通过逐条注入内容审计进一步定位。
