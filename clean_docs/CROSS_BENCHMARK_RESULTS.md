# Trace2Tower 与 SkillX 跨任务实验结论

本文只汇总当前可用于方法叙事的冻结结果。所有执行模型均为
`deepseek-v4-flash`，每项结果均覆盖对应 manifest 的全部唯一任务，且只统计正式
`results.jsonl`。

## 实验口径

主结果使用固定任务集合、`temperature=0` 和每题一次正式执行。表格只显示任务级率与
均值；任务数量和完整计数保留在主报告的实验设置与机器可读结果中。配对 bootstrap
以任务为重采样单位，只衡量任务组成不确定性，不代表多次独立运行方差。

## 任务性质与方法定位

| 数据集      | 任务性质                          | 图中需要保留的信息                | 有效技能形式                                |
|----------|-------------------------------|--------------------------|---------------------------------------|
| ALFWorld | 低分支、长程、显式实体与动作前置条件、二值完成       | 实体绑定、事件顺序、状态变换、放置或检查的完成链 | 从成功事件图发现多个可检索的长程 High 模式，局部技能补充动作前置条件 |
| WebShop  | 高分支、商品高度异质、属性逐页显露、允许部分 reward | 约束搜索、候选证据、选项完成、价格检查和条件恢复 | 从图主干直接归纳一张紧凑全局 High；商品事实只从当前页面读取      |

两者共享“领域事件抽取、成败图建模、层级关系发现”的方法契约，但不强迫图节点使用
同一种领域表示。ALFWorld 的任务类型和变换链在任务目标中可观测，可以检索不同 High；
WebShop 是否需要恢复只有看到搜索结果或商品页后才可观测，因此恢复模式应作为全局
High 内的条件分支，而不是另一张开局检索卡。

## SkillX 实际契约

| 数据集      | 构建库                                         | 运行时注入                                                | rewrite                                                  |
|----------|---------------------------------------------|------------------------------------------------------|----------------------------------------------------------|
| ALFWorld | 850 条全局成功轨迹，224 个 Plan、19 个 Function；无任务族标签 | Plan Top-3 后使用首个；逐步骤 Function Top-4、去重、最多 8 个；开局注入一次 | 主表使用冻结 no-rewrite；开源 rewrite-and-inject 另作诊断 |
| WebShop  | P100 成功池，51 个 Plan、2 个 Function             | Plan Top-3 后使用首个；逐步骤 Function Top-4、去重、最多 8 个；开局注入一次 | 官方实际入口使用 GPT-5.4 rewrite；另保留同 provider 的 no-rewrite 机制消融 |

ALFWorld 主表中的 SkillX 与 Trace2Tower 不共享 rewrite 后处理。SkillX 主表保留旧
no-rewrite 运行；开源 `plan_with_skill` 诊断会直接注入 rewritten Plan 和选中的
Function。Trace2Tower 使用自己的结构化 `budgeted_v2` rewrite 合同。

Trace2Tower WebShop 主结果不使用 rewrite，因为改写模型无法从任务文本推断尚未打开的商品页面事实。

## ALFWorld

### Valid-unseen

| 方法              |        成功率 |      平均步数 |   平均无效动作 | 平均输入 token | 平均 context 字符 |
|-----------------|-----------:|----------:|---------:|-----------:|--------------:|
| No-Skill        |     52.99% |     14.84 |     0.54 |     45,677 |             0 |
| Expert-Crafted Skills | 76.12% | 11.49 | 0.37 | **38,889** | 3,286 |
| SkillX no-rewrite |   81.34% |     11.69 |     0.49 |     64,651 |        13,092 |
| **Trace2Tower High-only** | **85.82%** | **10.50** | **0.22** | **37,410** |     **1,960** |

SkillX 相对 No-Skill 提高 `28.36` 个百分点，43 个任务由失败转为成功、5 个反向，
McNemar exact `p=1.37e-8`。Trace2Tower High-only 相对 SkillX 提高 `4.48` 个百分点，16 个
任务正向、10 个反向，McNemar exact `p=0.327`；这是正向点估计，不作显著性优越主张。

该 81.34% 是冻结的 ALFWorld no-rewrite 运行。SkillX 开源仓库在
`plan_with_skill` 模式下会把 rewritten plan 用于 skill 检索，并继续直接注入最终
system prompt；同集开启该路径的诊断结果为 67.91%。因此主表保留较早结果时必须带
`no-rewrite` 标签，不能声称它代表开源仓库的默认 rewrite 行为。

Expert-Crafted Skills 相对 No-Skill 提高 `23.13` 个百分点，证明人类领域先验是实质性
强基线。Trace2Tower High-only 相对它再提高 `9.70` 个百分点，配对 bootstrap 95% 区间为
`[+2.24,+17.16]` 个百分点，McNemar exact `p=0.0241`。这说明自动图归纳恢复了固定
人工策略没有完整表达的长程组合与失败对比结构。

这组结果支持 ALFWorld 的方法定位：完整长程阶段结构对执行具有直接价值。SkillX 的
Plan + Function 已显著提高成功率，Trace2Tower 从成败事件图归纳出的典型成功模式在
更短 context 下进一步减少步数和无效动作。包含 Mid 自筛选的 88.06% Full 运行保留为
case study，不用于本文件的主效果比较。

## WebShop

### 冻结结果

两个集合均未用于参数选择。本文统一编号为 `Test-1` 和 `Test-2`：`Test-1` 对应
仓库中的 `validation` manifest，`Test-2` 对应仓库中的 `TestA` manifest。

| split  | 方法                       | mean reward |     满分率 |     平均步数 |   平均无效动作 | 平均输入 token | 平均 context 字符 |
|--------|--------------------------|------------:|--------:|---------:|---------:|-----------:|--------------:|
| Test-1 | No-Skill                 |     0.65235 |     47% |     7.69 |     0.30 |     19,572 |             0 |
| Test-1 | SkillX no-rewrite        |     0.66285 |     47% |     8.07 |     0.39 |     31,366 |         6,681 |
| Test-1 | SkillX rewrite           |     0.68427 |     48% |     7.95 |     0.29 |     34,044 |         6,758 |
| Test-1 | Expert-Crafted Skills    |     0.70085 |     50% | **6.22** | **0.06** | **15,526** |         2,552 |
| Test-1 | **Trace2Tower 图主干 High** | **0.70402** | **49%** | **6.82** | **0.19** | **17,298** |     **1,193** |
| Test-2 | No-Skill                 |     0.68075 | **51%** |     7.98 |     0.36 |     21,388 |             0 |
| Test-2 | Expert-Crafted Skills    |     0.68875 |     50% | **6.46** | **0.06** | **18,642** |         2,552 |
| Test-2 | SkillX no-rewrite        |     0.70627 |  49.33% |     7.04 |     0.31 |     25,009 |             - |
| Test-2 | SkillX rewrite           |     0.66490 |     49% |     8.54 |     0.31 |     38,834 |         6,742 |
| Test-2 | **Trace2Tower 图主干 High** | **0.71432** |     49% | **6.39** | **0.15** | **16,290** |     **1,193** |



SkillX rewrite 的作用具有明显 split interaction：启用 rewrite 时 Test-1 较强、Test-2
下降；关闭 rewrite 时方向相反。它说明 WebShop 的困难不是缺少一份更具体的商品计划，
而是任务开始时没有足够商品证据支撑实例化改写。

Trace2Tower 图主干 High 在两个 split 都高于 No-Skill，并把 context 压到 1,193
字符。Test-1 reward 差值区间高于 0；Test-2 仍是方向性结果。它没有稳定提高二值满分
率，但提高了部分匹配 reward，同时减少执行步数、无效动作和输入 token。这与 WebShop
按连续 reward 评价、搜索分支高度同构的任务性质一致。

Expert-Crafted Skills 在 Test-1 和 Test-2 分别取得 `0.70085` 和 `0.68875`，均高于
各自 No-Skill 点估计，并在两组上保持最短步骤、最低无效动作和最低输入成本。Test-1
相对 No-Skill 的 reward 差为 `+0.04850`，95% 区间 `[-0.00617,+0.10667]`；相对
SkillX rewrite 为 `+0.01658`，区间 `[-0.01875,+0.05100]`。这证明固定人类先验是
可迁移的强基线，但它与自动图方法的点估计差距较小，当前样本量不足以支持显著优劣
结论。人工规则的明确优势是执行成本，图方法的潜在优势是 reward 上限和状态适应性。

## 跨任务结论

| 结论维度              | ALFWorld                                       | WebShop                                          |
|-------------------|------------------------------------------------|--------------------------------------------------|
| SkillX 的主要价值      | 检索到端到端 Plan，并用 Function 落实对象、前置条件和阶段数据流        | 提供完整购物计划，但 rewrite 受未观测商品状态限制，跨 split 不稳定        |
| Trace2Tower 的主要价值 | 从成败事件图归纳多个典型长程成功模式                             | 从异质商品轨迹的图主干直接归纳紧凑泛化 High                         |

因此，Trace2Tower 不应被表述为在所有环境中部署同一种技能层级数量。统一点是从成败
事件图中发现稳定可执行结构；最终提升为多少张 High、是否需要 Mid，取决于结构边界能否
在任务开始时被观测并可靠检索。
