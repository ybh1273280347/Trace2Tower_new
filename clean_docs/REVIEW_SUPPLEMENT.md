# Review 补充说明

日期：2026-07-23

## 1. 随机性边界

ALFWorld 对每个 `sample_id` 加载固定 game 文件；WebShop 使用本地冻结商品与搜索数据。环境状态本身不随会话变化，但远程模型 API 即使在 `temperature=0` 下仍非严格确定。正文保留冻结主实验的 Full 结果 88.06%，并将 High-only 作为同一 Trace2Tower 的运行时变体。后续新 API 供应时段内，No-Skill、SkillX 和 Trace2Tower 均明显低于冻结主实验；Trace2Skill 两个变体与 ExpeL 则基本保持原水平。Trace2Tower 继续追加三次后得到 68.66%、70.15% 和 64.93%，当前时段六次均值为 72.26%，样本标准差为 5.59 个百分点，没有向 80% 收敛。关闭 Mid 的受控运行时变体得到 70.15%、70.15% 和 66.42%，均值 68.91%，样本标准差 2.15 个百分点：方差收缩，但均值没有恢复。该结果说明供应端行为变化真实存在，但影响具有方法敏感性；Trace2Tower 的 plan rewrite 与分层注入路径对当前服务状态表现出额外敏感性。因此，这批运行单列记录，不进入主表均值，也不替换构建消融和运行时变体主表点估计。配对 bootstrap 只表示任务组成不确定性，不代表 API 重复运行方差。

## 2. WebShop 跨分区

同一冻结 P100 Tower 在第二个 100 题分区上的结果为：

| 方法 | Mean reward | 满分率 |
|---|---:|---:|
| No-Skill | **0.68075** | 51% |
| Trace2Tower | 0.67125 | **52%** |
| 差值 | −0.00950 | +1 pp |

两组结果的 100 个 `sample_id` 完全一致。连续 reward 与满分率方向相反，支持“跨分区收益不稳定”的边界描述，不支持在该分区声称稳定提升。本轮没有新增 WebShop 实验。

## 5. ALFWorld Full 主实验的失败与负迁移

review 中“12 个失败”的前提不准确。主实验 r0 为 118/134，因此有 16 个失败；三个既有执行器共同失败 7 个，而不是 12 个。主实验的 16 个失败按任务族分布如下：

| 任务族 | 失败数 |
|---|---:|
| Look in light | 1 |
| Pick & place | 2 |
| Clean & place | 1 |
| Cool & place | 3 |
| Heat & place | 5 |
| Pick two & place | 4 |

16/16 均正常召回 3 个 High 和 2–4 个 Mid，因此不是“没有 High 覆盖”。逐轨迹检查可归为三类：7 条在找到目标物前耗尽搜索预算；5 条已取得对象或完成部分变换，但未在 20 步内放置完成；4 条出现变换或恢复顺序偏离，例如未真正执行 clean/cool，或加热后转向无关位置。失败集中在对象定位、变换完成检查和多对象收尾，而不是检索接口缺失。

No-Skill 独赢共 4 条，分布为 2 条双对象放置、1 条普通放置、1 条清洁后放置。Tower 版本全部走满 20 步；对应 No-Skill 成功轨迹分别使用 16、20、11 和 17 步。三条与对象搜索或双对象收尾有关，一条遗漏了清洁动作。这说明少量负迁移主要来自指导改变搜索/完成顺序，不能归因于无效动作堆积，因为其中 3 条 Tower 轨迹没有无效动作。

## 6. Full 与 High-only 运行时变体比较

State A 中 Full 相对 High-only 为 7 胜、4 负、123 平，净差 +2.24 pp，`p=0.549`。该结果是两个预定义运行时变体的基线比较，不作为 Mid 的固定正向边际结论。State B/C 的逐题翻转与 bootstrap/McNemar 统计见 `clean_docs/API_STATE_ROBUSTNESS_CN.md`。

## 7. WebShop 成本收益分组

已有验证集的逐题配对结果为 16 条 reward 改善、9 条变差、75 条不变：

| 分组 | N | 平均 reward 差 | 平均输入 token 差 | 平均步骤差 |
|---|---:|---:|---:|---:|
| 改善 | 16 | +0.61875 | −8,358 | −3.56 |
| 变差 | 9 | −0.40648 | +63,025 | +10.33 |
| 不变 | 75 | 0 | +24,093 | +2.47 |

收益任务上 Tower 反而更快结束并减少 token；总体成本上升主要来自 reward 不变或变差的任务。这个结果支持未来研究选择性注入，但当前主方法没有基于测试题进行门控或选优。
